## Goals
In this tutorial, the goal was to understand how to send a patch by email using git. This time, we learned how to configure the email we'll be using to send these patches as well as how to configure 
an App Password, which we'll be using every time we send a patch.

## Problems and Difficulties
In this tutorial, my only problem was that I needed to change emails. Before, I was using my @usp email, which was linked to my GitHub account, but I had to change it to my personal email. That 
happened because, to send the patches, we'll need an App Password that is only possible to get if your Google account has two-factor authentication (2FA) enabled. However, the @usp email doesn't 
allow us to configure 2FA, so I needed to change to my personal email to be able to get this App Password.

Some other small problems I had were:
- Wasting time trying to configure 2FA on my @usp email before I realized that I couldn't (maybe it is possible, but I didn't find out how).
